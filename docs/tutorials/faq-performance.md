# 性能和存储常见问题

> 优化 nowen-note 的性能和存储使用。

---

## 性能

### Q：笔记多了之后变慢？

nowen-note 使用 SQLite 数据库，性能通常很好。如果变慢：

1. 执行数据库优化：⚙️ 设置 → 数据管理 → 数据库 → 优化
2. 清理回收站
3. 清理版本历史
4. 清理孤儿附件

### Q：搜索变慢？

首次搜索需要建立索引。后续搜索会很快。如果仍然慢：

1. 检查数据库大小
2. 执行数据库优化

### Q：编辑器卡顿？

1. 检查笔记内容是否特别长（超过 10 万字）
2. 尝试切换编辑器模式
3. 关闭浏览器其他标签页释放内存

---

## 存储

### Q：数据库文件在哪里？

```
data/nowen-note.db
```

### Q：数据库太大了？

1. 清理回收站中的笔记
2. 清理版本历史
3. 清理孤儿附件
4. 执行数据库优化

### Q：如何查看存储使用情况？

⚙️ 设置 → 数据管理，可以查看：

- 数据库大小
- 附件大小
- 备份大小
- 总占用空间

---

## 备份与恢复

### Q：备份文件太大？

备份文件包含完整数据库。几万篇笔记的数据库通常只有几十 MB。

### Q：如何减小备份大小？

1. 先清理不需要的数据（回收站、版本历史）
2. 再执行备份

---

## 优化建议

### 日常维护

- 定期清理回收站
- 定期清理版本历史
- 定期清理孤儿附件

### 定期检查

- 每月检查一次存储使用情况
- 每季度执行一次数据库优化
- 定期测试备份恢复
